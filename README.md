# mobile_dev

В ходе изучения мобильной разработки я выполняла различные тренировочные задания и собирала небольшие кроссплатформенные приложения. Разрабатывала в среде VS Code при помощи Flutter, тестировала в веб-среде (Chrome), либо с использованием Android Studio. Приложения, использующие сторонние API (api_1, api_2) могут быть капризны в отношении среды тестирования и нестабильно работать в какой-либо из них.
## Примененные технологии

Я разрабатывала интерфейс (material design под android), связывала между собой несколько страниц и применяла навигацию, использовала в своих приложениях несложные математические вычисления, правила и условия, добавляла обработчики событий для кнопок, таймеры, работала с библиотеками, добавляла api (картинки, новости).  

### Как протестировать:
Чтобы протестировать, вы можете скачать какой-либо проект целиком и запустить его в вашей среде разработки (для лучшей совместимости рекомендую использовать те инструменты, которые я использовала при разработке), однако проекты имеют большой вес и занимают много места на диске. Чтобы обойти это ограничение, вы можете скопировать код файла main.dart интересующего вас проекта и запустить его в "песочнице" - [zapp.run](https://zapp.run/) или [dartpad.dev](https://dartpad.dev/).

Пример пути к файлу main.dart: "mobile_dev\api_1\lib\main.dart".

Обращайте внимание, что в некоторых проектах есть папка assets, которая содержит необходимые реурсы проекта (например, картинки).

Отдельно хотелось бы обратить внимание на зависимости, без которых проект корректно работать не будет. В песочнице вам не придется задумываться об этом так же, как при разработке в полноценной среде. В файлах pubspec.yaml вы сможете узнать подробнее о том, какие ассеты и библиотеки я использовала в каждом из проектов. 

Для удобства я добавила файлы .apk, чтобы у вас была возможность скачать каждое из приложений на ваше android-устройство. Приложения безопасные и ваши данные не собирают.
